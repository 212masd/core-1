## 一、文档概述
本文档用于详细说明、解释并指导使用者安全、正确地运行你自行编写的 **AutoHotkey v2 自动化脚本**。该脚本主要面向浏览器多标签页批量操作、固定坐标点击、鼠标位置记录、快捷键触发输入等自动化场景设计，适用于需要重复切换标签、点击固定界面按钮、批量输入固定文本的重复性工作流程。脚本通过快捷键方式触发不同功能模块，操作轻量化、响应迅速，可大幅减少手动重复操作成本，提升操作效率。

## 二、脚本运行环境
### 2.1 AutoHotkey 版本要求
脚本开头明确标注：
```
#Requires AutoHotkey v2.0
```
这表示该脚本**仅支持 AHK v2 版本**，无法在 AHK v1 环境下运行。若用户安装的是 v1 版本，脚本启动时会直接报错，无法执行任何功能。

### 2.2 系统环境
脚本基于 Windows 系统编写，依赖系统鼠标坐标、窗口消息机制、键盘模拟输入机制，可在 Windows 10 / Windows 11 正常运行。

### 2.3 坐标模式
```
CoordMode "Mouse", "Screen"
```
该语句设置鼠标坐标以**整个屏幕**为基准，而非以当前活动窗口为基准。这意味着脚本记录和点击的坐标是**绝对屏幕坐标**，不会随窗口移动、缩放而自动偏移。

## 三、全局变量说明
脚本中定义了一组全局变量，用于存储鼠标坐标，实现跨快捷键的数据传递：
```
global clickX := 0
global clickY := 0
```

### 3.1 clickX
存储目标点击位置的**横向屏幕坐标**，初始值为 0。
该变量仅通过 F4 快捷键赋值，其他功能（如 F7）读取该值执行点击。

### 3.2 clickY
存储目标点击位置的**纵向屏幕坐标**，初始值为 0。
若 clickX 和 clickY 均为 0，F7 会判定为未记录坐标并弹出提示，避免无效点击。

## 四、脚本核心功能与快捷键详解
### 4.1 F9 —— 单次标签页切换
功能：模拟按下一次 `Ctrl + Tab`，切换到下一个浏览器标签。
执行逻辑：
1. 按下 Ctrl 键
2. 短暂延时保证按键稳定
3. 按下 Tab 键
4. 释放 Ctrl 键
5. 弹出提示“已切换1个标签页”，2 秒后自动消失

特点：轻量、快速，适用于手动配合脚本进行单步标签切换。

### 4.2 F8 —— 批量循环切换标签（40 轮）
功能：循环执行 40 次 `Ctrl + Tab` 切换标签操作。
执行逻辑：
1. 循环 40 次
2. 每次模拟一次完整 Ctrl+Tab
3. 每次切换后延时极长时间（940000ms）
4. 实时显示当前切换次数
5. 全部完成后提示“20次标签切换完成”

注意：
脚本内写的是 `Loop 40`，但提示文字写的是“20次”，存在文字与循环次数不一致问题。
另外，`Sleep 940000` 接近 **15 分钟**，实际使用中会造成极长时间停顿。

### 4.3 F3 —— 固定坐标点击 + 自动输入文本 + 回车
功能：执行一组固定坐标点击与文本输入，适用于固定界面固定按钮的固定操作流程。
执行步骤：
1. 点击固定坐标 (2496, 56)
2. 等待 200ms
3. 点击固定坐标 (2262, 1298)
4. 等待 900ms
5. 自动输入固定文本：
```
请把主播主的语言表达转化为文字生成文档文件以及对应的时间关系。回答不要展示文件，只给我 TXT 文件就行。
```
6. 等待 900ms
7. 按下回车键提交

特点：
完全写死坐标与文本，适用于界面布局永不变化的场景。
缺点：分辨率、缩放、窗口位置一旦改变，点击位置立即偏移。

### 4.4 F2 —— 单次固定坐标点击
功能：点击固定坐标 (2371, 633)，无其他操作。
用途：快速点击某个固定按钮，例如输入框、发送按钮、下拉框等。

### 4.5 F4 —— 记录当前鼠标坐标
功能：获取鼠标当前屏幕坐标，存入全局变量 clickX、clickY。
执行逻辑：
1. 获取当前鼠标 X、Y 坐标
2. 存入全局变量
3. 弹出提示显示坐标
4. 3 秒后提示消失

该功能为 F7 提供点击目标，是动态坐标功能的基础。

### 4.6 F6 —— 单次标签切换（精简版）
功能与 F9 几乎一致，模拟一次 Ctrl+Tab 切换标签。
与 F9 的区别：
- 延时更短
- 提示文案略有不同
- 代码结构更简洁

### 4.7 F7 —— 循环 40 次：切换标签 + 点击记录坐标
功能：批量自动化切换标签并点击目标位置，是脚本核心批量功能。
执行逻辑：
1. 判断 clickX 和 clickY 是否为 0
   - 若为 0 → 提示“请先按F4记录位置”并退出
2. 开始 40 次循环
3. 每次循环：
   - 切换一次标签页
   - 短暂延时确保切换完成
   - 点击之前 F4 记录的坐标
   - 等待 2 秒
   - 显示当前执行次数
4. 循环结束后提示完成

适用场景：
每个标签页都有相同位置按钮，需要逐个标签点击。

### 4.8 F12 —— 安全退出脚本
功能：弹出提示后彻底退出脚本，释放所有快捷键，避免后台继续运行。

## 六、脚本潜在缺陷、风险与注意事项
### 6.1 固定坐标强依赖屏幕布局
F3、F2 使用写死坐标，一旦：
- 屏幕分辨率改变
- 系统缩放 100%/125%/150%
- 浏览器窗口移动、最大化/还原
- 网页滚动
坐标立即失效，会点到空白或其他按钮。

### 6.2 循环次数与提示文案不一致
F8 中 Loop 40，但提示显示“20次”，容易造成误判。

### 6.3 F8 延时过长
Sleep 940000 毫秒，实际使用中几乎无法正常完成循环。

### 6.4 无窗口判断机制
脚本不判断当前窗口是否为浏览器，切换到其他窗口时依然会执行点击与输入，可能导致：
- 在文档里乱输入文字
- 误点系统按钮
- 删除文件、关闭程序

### 6.5 无异常恢复机制
若某次切换卡顿、页面卡死，脚本不会中断，会继续盲目执行。

### 6.6 无防误触机制
不小心按到 F3/F7/F8 会立即执行大量操作，难以中断。

### 6.7 无点击有效性判断
脚本只执行点击，不判断是否点中、页面是否加载完成。

## 七、常见问题与解决方案
### 7.1 脚本运行后没反应
- 检查是否为 AHK v2
- 检查是否以管理员身份运行（某些窗口需要权限）
- 检查是否被安全软件拦截

### 7.2 F7 提示未记录坐标
需先按 F4 获取坐标，再执行 F7。

### 7.3 点击位置偏移
- 重新按 F4 记录新坐标
- 保持窗口位置、缩放不变

### 7.4 文本输入乱码或不完整
- 增加 Sleep 时间
- 确保当前焦点在输入框
